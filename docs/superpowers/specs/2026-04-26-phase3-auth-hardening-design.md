This spec is the **incremental design for Phase 3** on top of the parent backend design. The parent spec already settled most architecture (token types, table shapes, endpoint shapes, rate-limit table, cookie attributes, CSRF stance). This document records the boundary decisions made on 2026-04-26 and the concrete artefacts P3 must deliver.

---

## 1. Goal

Bring admin auth from "password + 15-min JWT" (Phase 1 baseline) to the full design described in the parent spec §6:

- Refresh tokens with rotation (Redis-backed)
- TOTP 2FA (challenge-based two-step login) with **recovery codes**
- Magic-link login (DB-backed single-use, dev-mode email = stdout log)
- API tokens (bcrypt-hashed, raw-once, **read/write scopes**)
- Rate limiting (self-rolled Redis limiter, 3 key dimensions)

After P3, every admin auth flow described in the parent spec is implemented and reachable end-to-end; only the email transport (SMTP) and worker queue (ARQ) are deferred to P5.

---

## 2. Out of scope (explicit)

The following are **deliberately deferred** so this phase has a clean boundary:

| Item | Deferred to | Reason |
|---|---|---|
| Real SMTP (or any real email transport) | P5 | ARQ + SMTP land together |
| ARQ worker queue | P5 | Worker scaffold belongs with integration phase |
| Resource-grained API token scopes (e.g. `posts:write`) | — | YAGNI for single-author blog |
| Access-token blacklist on logout | — | 15-min TTL is short enough; refresh deletion + password change cover the threat |
| WebAuthn / passkeys | — | TOTP is sufficient for v1 |
| SMS or email-OTP 2FA | — | TOTP only |

## 3. Data model changes

One Alembic revision: `0002_auth_phase3`. No changes to existing tables — only additions.

### 3.1 `magic_links`

```sql
CREATE TABLE magic_links (
    token_hash      VARCHAR(64)  PRIMARY KEY,         -- sha256 hex
    account_id      INTEGER      NOT NULL REFERENCES accounts(id) ON DELETE CASCADE,
    expires_at      TIMESTAMPTZ  NOT NULL,
    consumed_at     TIMESTAMPTZ,                      -- NULL = unused
    requested_ip    VARCHAR(45),
    user_agent      VARCHAR(256),
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT now()
);
CREATE INDEX ix_magic_links_expires_at ON magic_links (expires_at);
```

### 3.2 `api_tokens`

```sql
CREATE TABLE api_tokens (
    id              INTEGER      PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    name            VARCHAR(64)  NOT NULL,            -- human label, e.g. "github-actions"
    scope           VARCHAR(8)   NOT NULL,            -- 'read' | 'write'
    token_hash      VARCHAR(64)  NOT NULL UNIQUE,     -- sha256 hex (see §6.3 deviation note)
    last_used_at    TIMESTAMPTZ,
    revoked_at      TIMESTAMPTZ,
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT now(),
    CONSTRAINT ck_api_tokens_scope CHECK (scope IN ('read', 'write'))
);
CREATE INDEX ix_api_tokens_revoked_at ON api_tokens (revoked_at) WHERE revoked_at IS NULL;
```

### 3.3 `tfa_recovery_codes`

```sql
CREATE TABLE tfa_recovery_codes (
    code_hash       VARCHAR(64)  PRIMARY KEY,         -- sha256 hex
    account_id      INTEGER      NOT NULL REFERENCES accounts(id) ON DELETE CASCADE,
    used_at         TIMESTAMPTZ,
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT now()
);
CREATE INDEX ix_tfa_recovery_codes_account_id ON tfa_recovery_codes (account_id);
```

> **Note**: `accounts.tfa_secret_encrypted`, `accounts.tfa_enabled`, `accounts.magic_link_enabled` already exist (Phase 1). P3 starts populating them.

## 4. Token / credential matrix

| Type | Encoding | TTL | Storage | Rotation |
|---|---|---|---|---|
| Access | HS256 JWT (`sub`, `jti`, `iat`, `exp`) | 15 min | stateless | new on every issue |
| Refresh | 32-byte opaque (URL-safe base64) | 30 d | Redis `refresh:{sub}:{jti}` | rotated on every `/refresh` (old jti deleted) |
| Magic-link | 32-byte opaque, **sha256** in DB | 15 min | DB `magic_links` | single-use (`consumed_at`) |
| API token | 32-byte opaque (`tk_` + base64), **sha256** in DB | revoke-only | DB `api_tokens` | manual via DELETE |
| 2FA challenge | opaque `jti` | 5 min | Redis `2fa:{challenge}` (value = `account_id`) | single-use (delete on success) |
| Recovery code | 8 × `XXXX-XXXX` (Crockford base32, 4+4 chars + dash) **sha256** in DB | one-time | DB `tfa_recovery_codes` | regenerable (replaces all 8) |

**Key encryption** (`tfa_secret_encrypted`): use AES-GCM with `secrets_key` (already in settings). Service module `app/services/secret_box.py` exposes `encrypt(str) -> bytes`, `decrypt(bytes) -> str`. Re-used later for OAuth tokens etc.

## 5. Endpoints

All under `/api/admin` prefix, mounted in `app/routers/admin/`. Schemas live in `app/schemas/auth.py` (extended).

### 5.1 Auth (`routers/admin/auth.py` — extended)

| Method | Path | Body | Response | Rate-limited |
|---|---|---|---|---|
| POST | `/auth/login` | `{email, password}` | `{access, refresh:cookie}` **or** `{tfa_required:true, challenge}` | yes (IP) |
| POST | `/auth/2fa` | `{challenge, code}` *(TOTP **or** recovery code)* | `{access, refresh:cookie}` | yes (challenge) |
| POST | `/auth/refresh` | — *(uses cookie)* | `{access, refresh:cookie}` *(rotated)* | no |
| POST | `/auth/logout` | — | `204` | no |
| POST | `/auth/magic-link` | `{email}` | `202` | yes (email) |
| GET | `/auth/magic-link/verify?t={token}` | — | `{access, refresh:cookie}` | no (token-bound) |

**Cookie attrs**: `Secure` (in prod env), `HttpOnly`, `SameSite=Lax`, `Path=/api/admin/auth`, `Max-Age=2592000`.

**`/auth/2fa` code disambiguation**: codes are length-distinguished (TOTP = 6 digits, recovery = 9-char `XXXX-XXXX`). pyotp `verify()` uses ±1 30-second step window. Match TOTP first; if length matches recovery format, look up `tfa_recovery_codes` and mark `used_at` on success. On total miss, increment challenge attempt counter; 5 fails → delete challenge → 401.

### 5.2 Account (`routers/admin/account.py` — new)

All require `current_admin` dependency.

| Method | Path | Body | Response |
|---|---|---|---|
| POST | `/account/2fa/setup` | — | `{secret, otpauth_uri, qr_svg}` *(secret stored encrypted but `tfa_enabled=false` until verify)* |
| POST | `/account/2fa/enable` | `{code}` | `{recovery_codes: [8]}` *(raw, shown once)* |
| DELETE | `/account/2fa` | `{current_code}` | `204` *(also wipes recovery codes)* |
| POST | `/account/2fa/recovery-codes/regenerate` | `{current_code}` | `{recovery_codes: [8]}` |
| PATCH | `/account/magic-link` | `{enabled: bool}` | `200 {magic_link_enabled}` |

### 5.3 API tokens (`routers/admin/api_tokens.py` — new)

| Method | Path | Body | Response |
|---|---|---|---|
| GET | `/api-tokens` | — | `[{id, name, scope, last_used_at, created_at}]` *(no token_hash)* |
| POST | `/api-tokens` | `{name, scope: 'read'\|'write'}` | `{id, name, scope, token: 'tk_...'}` *(raw, **shown once**)* |
| DELETE | `/api-tokens/{id}` | — | `204` *(sets `revoked_at`)* |

### 5.4 API-token-authenticated requests

A new dependency `current_api_token` accepts `Authorization: Bearer tk_<raw>`:

1. Lookup all non-revoked rows by trying bcrypt verify against each `token_hash` (single-author blog → typically ≤5 tokens).
2. On match: update `last_used_at`, return token row.
3. `require_scope('write')` rejects `read` tokens with 403.

Public endpoints (`/api/posts`, `/api/site`, etc.) ignore api-tokens; only `/api/admin/*` consumes them. `current_admin` is extended to accept either:
- a JWT access token (session admin), **or**
- an API token whose scope satisfies the endpoint's `require_scope`.

### 6.1 `rate_limit.py` (Redis-backed)

```python
class RateLimited(Exception):
    def __init__(self, retry_after: int) -> None:
        self.retry_after = retry_after

async def hit(
    redis: Redis,
    key: str,
    *,
    limit: int,
    window_sec: int,
) -> None:
    """Atomic INCR + EXPIRE. Raises RateLimited if count > limit."""
    pipe = redis.pipeline()
    pipe.incr(key)
    pipe.expire(key, window_sec, nx=True)  # only set TTL on first hit
    count, _ = await pipe.execute()
    if count > limit:
        ttl = await redis.ttl(key)
        raise RateLimited(retry_after=max(ttl, 1))
```

**Login lockout** (separate key):
- `rl:login:{ip}` — 5/min sliding (the limiter above)
- `rl:login_fails:{ip}` — counts only **failed** logins; at 10, set `rl:login_lock:{ip}` for 15 min; while locked, `/login` returns 429 immediately.
- On successful login, delete `rl:login_fails:{ip}`.

**3 enforced limiters**:

| Endpoint | Key | Limit |
|---|---|---|
| `POST /auth/login` | `rl:login:{ip}` | 5/min |
| `POST /auth/login` failures | `rl:login_fails:{ip}` | 10/15min → lock |
| `POST /auth/2fa` | `rl:2fa:{challenge}` | 5/challenge (TTL=challenge TTL=5min) |
| `POST /auth/magic-link` | `rl:mlink:{email}` | 3/hour |

Errors raise `RateLimited(retry_after=…)`, caught by global handler → `429` with `Retry-After` header and JSON `{"detail":"rate limited","retry_after":…}` (already wired in Phase 1 error envelope).

### 6.3 `totp.py`, `recovery_codes.py`, `api_tokens.py`

Standard wrappers. Each ≤80 LOC. Concrete signatures:

```python
# totp.py
def generate_secret() -> str                   # base32, 32 chars
def otpauth_uri(*, secret: str, email: str) -> str
def qr_svg(uri: str) -> str                    # SVG string via segno
def verify(secret: str, code: str) -> bool     # ±1 30-sec step window (pyotp default)

# recovery_codes.py
def generate_set() -> list[str]                # 8 × "XXXX-XXXX" Crockford
def hash_code(code: str) -> str                # sha256 hex
async def verify_and_consume(s: AsyncSession, *, account_id: int, code: str) -> bool

# api_tokens.py
def generate_raw() -> str                      # 'tk_' + 32B urlsafe b64
def hash_raw(raw: str) -> str                  # sha256 hex
async def verify_and_touch(s: AsyncSession, raw: str) -> ApiToken | None
def require_scope(scope: Literal['read','write']) -> Depends
```

> **Deviation from parent spec §3 / §6**: the parent spec specified bcrypt for `api_tokens.token_hash`. P3 uses **sha256** instead. Justification: api-token raw values are 32 bytes of CSPRNG output (≥256-bit entropy) — they are not user-chosen passwords, so bcrypt's work factor adds no protection against guessing while breaking O(1) lookup (bcrypt requires comparing the candidate against every stored hash). sha256-hex enables a single indexed `WHERE token_hash = $1` lookup. This deviation does not affect any external interface; the parent spec is updated implicitly via this child spec.

## 7. Dependency wiring

`app/deps.py` extends:

```python
async def current_admin(
    request: Request,
    s: AsyncSession = Depends(get_session),
    redis: Redis = Depends(get_redis),
) -> Account:
    """Accepts either a JWT access token or an API token in Authorization header."""
```

`current_admin` becomes the union; for endpoints that **must** be session admin (e.g. 2FA management), use `current_session_admin` (rejects api-tokens). For read-vs-write enforcement on api-tokens, use `Depends(require_scope('write'))` on the relevant operation.

## 12. Migration / rollout

P3 is purely additive — no changes to public API surface, no changes to existing admin endpoints. Deployment is a single `alembic upgrade head` plus restart. Rollback is `alembic downgrade -1`; admin can still log in with password (2FA is opt-in).
